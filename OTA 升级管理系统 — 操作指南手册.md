## 1 背景信息

OTA（Over-The-Air）升级管理系统是物联网平台的核心功能模块，用于对已部署在现场的物联网设备进行远程固件升级，无需现场烧录。

系统支持两种升级包类型：

*   **整包升级**：设备下载完整的目标版本固件包进行升级。适用于大多数常规升级场景，兼容性好，但固件包体积较大，对设备存储空间和网络带宽有一定要求。
    
*   **差分包升级**：设备仅下载当前版本与目标版本之间的差异部分（差分包）进行升级。固件包体积小、传输快、流量消耗低，但要求设备当前版本与差分包的基线版本严格匹配。**目前仅 4G 模组产线支持差分升级，其他产线暂不支持****。**
    

本系统支持以下核心能力：

*   **创建升级任务**：配置任务名称、升级包类型（整包/差分）、目标固件版本和任务所属大区，选择升级策略确定设备范围。
    
*   **灵活选择设备**：支持按版本筛选、文件批量导入、手动输入三种方式确定升级范围。
    
*   **预览与确认**：提交前预览任务配置和设备可升级性校验结果，确认无误后提交。
    
*   **监控升级进度**：实时查看每台设备的升级状态和整体进度。
    
*   **结束任务**：在发现问题时及时终止升级过程。

## 2 名词解释

| **名词** | **解释** |
| --- | --- |
| OTA 升级 | Over-The-Air 升级，即通过无线网络对设备进行远程固件更新的技术。 |
| 升级任务 | 一次针对特定设备集合的固件升级操作，包含升级包、设备范围、升级策略等配置。 |
| 整包升级 | 设备下载完整的目标版本固件包进行升级的方式。固件包体积较大，兼容性好。 |
| 差分升级 | 设备仅下载当前版本与目标版本之间的差异部分（差分包）进行升级的方式。包体积小，但要求设备当前版本与差分包基线版本严格匹配。**目前仅 4G 模组产线支持****。** |
| 整包（固件包） | 包含完整目标版本固件的二进制文件，任何低于目标版本的设备均可使用。 |
| 差分包 | 通过目标版本与特定基线版本的差分算法生成的文件，仅适用于运行该基线版本的设备。 |
| 升级策略 | 本次升级任务选择目标设备的方式：指定版本号升级、文件导入升级、手动导入升级。 |
| 指定版本号升级 | 通过设备当前固件版本号筛选目标设备的升级策略，支持"全部升级"、"指定版本"和"排除指定版本"三种版本过滤规则。 |
| 版本过滤规则 | 指定版本号升级策略下，确定升级设备范围的筛选方式：全部升级（自动选择所有低版本设备）、指定版本（手动勾选特定版本）、排除指定版本（手动排除特定版本）。 |
| 文件导入升级 | 通过上传包含设备标识列表的文件（CSV、Excel）批量导入目标设备的升级策略。 |
| 手动导入升级 | 通过手动输入设备标识逐一添加目标设备的升级策略。 |
| 任务状态 | 升级任务在其生命周期中所处的阶段，包括待审批、待执行、执行中、已完成、已结束。 |
| 任务周期 | 升级任务的计划执行时间范围，由开始时间和结束时间组成。开始时间必须晚于当前时间，任务周期不能超过 7 天。 |
| 产线负责人审批 | 指定版本号升级和文件导入升级策略创建的任务，提交后需经产线负责人审批通过后才能执行。手动导入升级策略无需审批。 |
| 设备升级状态 | 单台设备在升级过程中的状态，包括待升级、升级中、升级成功、升级失败、已终止。 |

## 3 前提条件

在使用 OTA 升级管理系统之前，请确认以下条件已满足：

:::
您已拥有平台账号并具备 OTA 升级管理的操作权限；

注册运维3.0平台账号步骤详见《[固件管理操作指南](https://alidocs.dingtalk.com/i/nodes/ZgpG2NdyVXrAGar3sbzDMx6X8MwvDqPk?utm_scene=team_space)》。

如需配置OTA系统权限可联系 $\color{#0089FF}{@汤彦珊}$ ；

已上传并发布至少一个固件版本（包含升级固件）。

如需使用**FOTA差分升级**，需确认：

设备所属产线为 **4G 模组产线**（其他产线暂不支持差分升级）。

目标固件版本和待升级的基线版本均已上传"用于差分的文件"。

已通过差分管理生成对应的差分包，且差分包状态为"生成成功"。

已有注册设备，且设备支持 OTA 远程升级。

设备的控制板/模组需要支持固件升级功能。
:::

## 4 查看升级任务列表

任务列表页是 OTA 升级管理系统的主入口页面，用于查看和管理所有已创建的升级任务。

### 操作步骤

1.  登录运维3.0平台。
    
2.  在左侧导航栏中，选择 **OTA 升级** > **任务列表**。
    
3.  系统加载并展示所有升级任务的列表。
    

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/b14265a8-c726-45cd-b8e4-0c0658bef4ae.png)

### 页面字段说明

| **字段** | **说明** |
| --- | --- |
| 任务名称 | 升级任务的名称，超过 20 字符截断显示，鼠标悬停可查看完整内容。 |
| 升级策略 | 本次升级任务选择目标设备的方式：指定版本号升级 / 文件导入升级 / 手动导入升级。 |
| 升级设备总数 | 纳入本次升级范围的设备总数。 |
| 任务起止时间 | 任务开始执行和结束的时间区间，格式如 "2026-04-01 10:00 ~ 2026-04-01 12:30"，未开始显示"—"。 |
| 升级成功率 | 升级成功设备数占升级设备总数的百分比，格式如 "85.0%"。 |
| 升级失败数 | 升级失败的设备数量。 |
| 任务所属大区 | 本次升级任务覆盖的设备所在地理大区。 |
| 创建时间 | 任务创建的时间，格式 "YYYY-MM-DD HH:mm:ss"。 |
| 操作 | 查看详情（始终显示）、结束任务（仅待执行或执行中状态显示）。 |

### 搜索与筛选

*   **搜索**：在搜索框中输入任务名称关键词，按回车触发，支持模糊匹配。
    
*   **状态筛选**：在状态下拉框中选择一个或多个任务状态进行筛选。
    
*   搜索和筛选条件可同时生效，取交集。清空条件后恢复展示全部任务。
    

### 分页与排序

*   默认每页 10 条，支持切换为 20、50、60、80、100 条每页。
    
*   默认按创建时间降序排列。
    
*   可排序字段：创建时间、升级设备总数。
    

### 快捷操作

*   点击任务【操作】列【详情】，跳转至任务详情页。
    
*   点击右上角 **新增任务** 按钮，进入新增升级任务创建流程。

## 5 新增升级任务

### 5.1 基本流程

新增升级任务在一个页面内完成所有配置。页面从上到下依次为：基础信息区域、设备选择区域、任务配置预览区域。填写完成后单击页面底部的 **预览并发布** 按钮即可创建任务；

```plaintext
Step 1：基础信息配置  ──▶  Step 2：选择目标设备  ──▶  Step 3：任务配置预览
     │                          │                          │
  填写任务名称              根据升级策略进入               预览全部配置信息
  选择升级策略              对应的设备选择页面：            校验设备可升级性
  选择升级包类型            · 指定版本号 → 版本筛选         确认并提交任务
  （整包/差分）            · 文件导入 → 上传文件
  选择目标固件版本          · 手动导入 → 输入设备标识
  选择任务所属大区

```

各策略之间支持切换，已填写的信息保留不丢失。

---

### 5.2 第一步：基础信息配置

在此步骤中，您需要配置升级任务的基础信息。升级包类型的选择会直接影响后续固件版本的可选范围和设备匹配逻辑。

#### 操作步骤

1.  在任务列表页，单击右上角的 **新增任务** 按钮。
    
2.  系统打开新增任务页面：**基础配置**（当前）→ 升级策略选择 → 配置预览。
    
3.  依次填写以下信息：
    
    | **参数** | **是否必填** | **说明** |
    | --- | --- | --- |
    | 任务名称 | 是 | 1-64 个字符，支持中文、英文、数字、下划线、中划线，不允许纯空格，不允许与已有任务重复。 |
    | 升级策略 | 是 | 下拉单选：**指定版本号升级**、**文件导入升级**、**手动导入升级**。选择后决定 Step 2 的设备选择页面（详见下方联动说明）。 |
    | 升级包类型 | 是 | 单选：**整包** 或 **差分**。差分仅 4G 模组产线支持差分包升级。选择后影响目标固件版本的可选范围（详见下方联动说明）。 |
    | 目标固件版本 | 是 | 从已发布的固件版本列表中选择。列表内容受"升级包类型"联动控制。 |
    | 任务所属大区 | 是 | 下拉单选，选择任务覆盖的设备所在地理大区。 |
    | 任务起止时间 | 是 | 设置任务的计划执行时间范围（开始时间 ~ 结束时间）。详见下方任务周期规则。 |
    
4.  确认信息无误后，可进行 固件版本过滤选择。
    

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/fd2d9e62-b5e3-4c8b-b319-475919b55ee9.png)

#### 升级策略与设备选择区域的联动规则

选择"升级策略"后，页面下方的设备选择区域实时切换为对应的操作界面：

| **升级策略** | **设备选择区域内容** | **适用场景** | **是否需要审批** |
| --- | --- | --- | --- |
| **指定版本号升级** | 展示版本筛选页面，支持"指定版本"和"排除版本"两种筛选方式 | 按设备当前固件版本批量筛选，适合大规模版本统一升级 | 需要审批 |
| **文件导入升级** | 展示文件上传页面，支持 CSV/Excel 批量导入设备列表 | 已有明确设备清单，适合从外部系统导出的设备列表 | 需要审批 |
| **手动导入升级** | 展示文本输入页面，支持逐行输入或粘贴设备标识 | 设备数量较少或需要精确指定，适合定向升级或测试升级 | 无需审批 |

#### 任务周期规则

任务周期用于设定升级任务的计划执行时间范围，由"开始时间"和"结束时间"组成。

| **规则** | **说明** |
| --- | --- |
| 开始时间 | 必须晚于当前时间。如果设置的开始时间早于或等于当前时间，系统提示"开始时间必须晚于当前时间"。 |
| 结束时间 | 必须晚于开始时间。 |
| 最大周期 | 任务周期（结束时间 - 开始时间）不能超过**7 天（168 小时）**。如果超过，系统提示"任务周期不能超过 7 天"。 |
| 时间精度 | 精确到秒，格式为 "YYYY-MM-DD HH:mm:ss"。 |
| 任务到期 | 任务周期结束时，如果仍有未完成升级的设备，系统自动将剩余待升级设备标记为"已终止"，任务状态流转为"已完成"。 |

:::
**注意**：

任务在开始时间到达后自动开始执行，无需手动触发。

任务在结束时间到达后自动停止下发新的升级指令，已开始升级的设备继续完成当前升级。

建议根据设备数量和网络条件合理设置任务周期，避免周期过短导致部分设备未能完成升级。
:::

#### 审批流程规则 

根据升级策略的不同，任务提交后的流转路径不同：

| **升级策略** | **提交后流程** | **说明** |
| --- | --- | --- |
| **指定版本号升级** | 发布 →**待审批** → 产线负责人审批通过 → 待执行 → 到达开始时间后自动执行 | 涉及批量设备，需产线负责人确认升级范围和影响 |
| **文件导入升级** | 发布 →**待审批** → 产线负责人审批通过 → 待执行 → 到达开始时间后自动执行 | 涉及批量设备，需产线负责人确认升级范围和影响 |
| **手动导入升级** | 发布 → 待执行 → 到达开始时间后自动执行 | 设备数量少且精确指定，无需审批，提交后直接进入待执行 |

**审批交互说明：**

*   选择"指定版本号升级"或"文件导入升级"策略时，单击"提交"按钮后，系统提示"任务已提交，等待产线负责人审批"，任务状态为"待审批"。
    
*   产线负责人在审批页面查看任务配置信息，可选择"通过"或"驳回"。
    
*   ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/020eca9e-3aa0-4a6a-a671-b896a9e39667.png)
    
*   审批通过后，任务状态流转为"待执行"，到达任务周期的开始时间后自动开始执行。
    
*   审批驳回后，任务状态流转为"已驳回"，创建人可在任务详情页查看驳回原因。如需重新升级，需创建新的任务。
    
*   选择"手动导入升级"策略时，单击"提交"按钮后，任务直接进入"待执行"状态，无需审批。
    

#### 升级包类型与固件版本的联动规则

升级包类型的选择直接决定了"目标固件版本"下拉列表中可选的版本范围：

| **升级包类型** | **目标固件版本可选范围** | **设备匹配逻辑** |
| --- | --- | --- |
| **整包** | 所有已上架且包含升级固件的版本 | 任何低于目标版本的设备均可接收整包升级，设备下载完整固件包。 |
| **差分包** | 仅展示已生成差分包且差分包状态为"生成成功"的版本 | 仅当设备当前版本与某个差分包的基线版本严格匹配时，设备才能接收差分升级。 |

**联动交互流程：**

1.  用户选择"升级包类型"为 **整包** → 系统加载所有已上架的固件版本列表。
    
2.  用户选择"升级包类型"为 **差分** → 系统仅加载存在可用差分包的固件版本列表。
    
    *   如果无可用的固件版本号差分包，下拉列表为空，无法进行差分升级。
        
3.  用户切换升级包类型时，已选择的目标固件版本自动清空，版本列表重新加载。
    

:::
**整包 vs 差分 — 如何选择？**

| **对比项** | **整包升级** | **差分包升级** |
| --- | --- | --- |
| **固件包大小** | 较大（完整固件） | 较小（仅差异部分） |
| **适用设备** | 所有低于目标版本的设备 | 仅当前版本与差分基线版本匹配的设备 |
| **支持产线** | 所有产线 | **当前仅【4G模组】产线支持** |
| **网络要求** | 带宽要求较高 | 带宽要求低 |
| **存储要求** | 设备需要有足够存储空间 | 存储要求低 |
| **版本要求** | 支持跨多个版本升级 | 仅支持特定版本间升级 |
| **推荐场景** | 常规升级，版本跨度大 | 4G模组小版本迭代 |
:::
> **注意**：所有必填项填写完成、设备选择完成且存在可升级设备后，页面底部的"提交"按钮才可点击。单击"取消"按钮会弹出确认对话框，确认后返回任务列表页。

---

### 5.3 第二步：选择升级策略

根据 Step 1 中选择的升级策略，系统自动展示对应的设备选择页面。以下分别说明三种升级策略的操作方式。

> **重要提示**：如果选择了 **差分升级包类型**，则在指定版本号升级模式下，版本列表中仅展示与已生成差分包的基线版本匹配的版本。不匹配的版本会进行过滤且不可勾选。

---

#### 5.3.1 升级策略一：指定版本号升级

通过设备当前固件版本号筛选目标设备，适用于按版本批量升级的场景。

##### （1）规则策略操作说明

1.  在"升级策略"中选择 **指定版本号升级**，页面下方自动展示版本筛选区域。
    
    ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/c253d035-e62b-41d5-86e1-f7f83c904216.png)
    
    > 如上图所示，选择"指定版本号升级"策略后，页面下方展示版本筛选区域。顶部可切换版本过滤规则“全部”"指定版本"和"排除版本"三种筛选方式，下方为版本列表，每项显示版本号和对应设备数量。
    
2.  系统加载所有已知固件版本列表，每项展示格式为："版本号（设备数量 台）"，如 "v1.0.0（1,234 台）"。
    
3.  选择【版本过滤规则】筛选策略规则（三选一）：
    
    *   **全部升级**（默认）**：**
        
        *   选择"全部升级"后，系统自动将所有低于目标版本的设备纳入升级范围，无需手动勾选版本。
            
        *   版本列表以只读方式展示所有版本及设备数量，供用户确认。
            
        *   目标固件版本（与基础信息中选择的版本相同）自动排除，不计入升级范围。
            
        *   适用场景：希望一次性将所有旧版本设备统一升级到目标版本。
            
    *   **指定版本**：选择一个或多个版本，系统将当前运行这些版本的设备纳入升级范围。
        
        *   适用场景：仅升级某些特定旧版本的设备，其他版本暂不升级。
            
        
        ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/191dbcb2-1e9b-4657-b5d9-af98bd0ca864.png)
        
        > 如上图所示，选择"指定版本"筛选方式后，勾选需要升级的版本，系统将当前运行这些版本的设备纳入升级范围。右侧实时显示匹配设备总数。
        
    *   **排除版本**：勾选一个或多个版本，系统将这些版本的设备排除在升级范围之外，其余版本的设备纳入升级范围。
        
        *   适用场景：大部分版本都需要升级，仅排除少数特定版本。
            
    
    ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/8ac4d4ed-f562-4e11-9821-4936499c487f.png)
    
    > 如上图所示，选择"排除版本"筛选方式后，勾选需要排除的版本，系统将这些版本之外的设备纳入升级范围。
    
4.  在版本列表中勾选一个或多个版本，系统实时计算并展示匹配设备总数。
    

##### （2）三种版本过滤规则对比

| **版本过滤规则** | **是否需要勾选版本** | **升级范围** | **适用场景** |
| --- | --- | --- | --- |
| **全部升级** | 否（自动选择） | 所有低于目标版本的设备 | 全量版本统一升级 |
| **指定版本** | 是（手动勾选） | 仅勾选版本对应的设备 | 仅升级特定旧版本 |
| **排除指定版本** | 是（手动勾选） | 除勾选版本外的所有设备 | 排除少数版本，其余全部升级 |

##### （3）差分升级时的版本列表联动

当升级包类型为"差分"时，版本列表的展示逻辑如下：

| **版本状态** | **展示样式** | **是否可勾选** | **说明** |
| --- | --- | --- | --- |
| 存在对应差分包（基线版本匹配） | 正常展示 | 可勾选 | 该版本设备可接收差分包升级 |

:::
**注意**：差分升级 + 全部升级规则时，系统仅将存在对应差分包的版本设备纳入升级范围，不存在差分包的版本设备自动排除。
:::

#### 5.3.2 升级策略二：文件导入升级

*   通过上传设备列表文件批量导入目标设备，适用于设备数量较大、已有明确设备清单的场景。
    

##### 操作说明

1.  在"升级策略"中选择 **文件导入升级**，页面下方自动展示文件上传区域。
    
    ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/8182dc48-7166-4f9b-b47c-06dc259168a7.png)
    
    > 如上图所示，选择"文件导入升级"策略后，页面下方展示文件上传区域。支持拖拽文件至虚线框内，或单击虚线框选择文件。区域下方展示文件格式要求和导入模板下载链接。
    
2.  将设备列表文件拖拽至上传区域，或单击上传区域选择文件。
    
3.  系统校验文件格式和大小，校验通过后自动开始解析，解析过程中展示设备总数。
    

##### 文件格式要求

| **属性** | **说明** |
| --- | --- |
| 支持格式 | CSV（.csv）、Excel（.xlsx、.xls） |
| 文件大小限制 | 单个文件不超过 5MB |
| 最大设备数 | 单个文件最多 20,000 个设备标识 |
| 编码要求 | CSV 文件支持 UTF-8 和 GBK 编码 |
| 文件结构 | 第一列为设备标识，第一行可为表头（系统自动识别并跳过） |

##### 设备标识校验规则

| **校验项** | **规则** | **失败原因** |
| --- | --- | --- |
| 格式校验 | 长度 1-64 字符，仅支持字母、数字、中划线、下划线 | "设备标识格式错误" |
| 存在性校验 | 设备标识在平台中已注册 | "设备不存在" |
| 归属校验 | 设备属于当前升级任务的适用范围 | "设备不属于当前升级范围" |
| 去重校验 | 同一设备标识出现多次时自动去重 | 不报错，提示去重数量 |

> **差分升级时的额外校验**：如果选择了差分升级包类型，系统会额外校验每台设备的当前固件版本是否与已有差分包的基线版本匹配。不匹配的设备标记为"无可用差分包"。

> **注意**：

*   仅有效设备纳入升级范围，无效设备不纳入。
    

*   可单击"重新上传"清除当前导入结果并重新选择文件。
    
*   建议先下载导入模板，按模板格式填写后上传。
    
*   Excel 文件包含多个 Sheet 时，仅读取第一个 Sheet。
    
*   文件中的空行自动跳过，设备标识前后空格自动去除。

## 7 结束升级任务

在任务执行过程中，如发现固件存在问题或需要终止升级，可通过"结束任务"操作立即停止升级流程。

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/fe899073-c727-4891-bee3-cec0824c36d5.png)

### 操作权限

| **当前任务状态** | **操作按钮** |
| --- | --- |
| 待执行 | 【结束任务】 |
| 执行中 | 【结束任务】 |
| 已完成 | 不展示 |
| 已结束 | 不展示 |

### 操作步骤

1.  在任务详情页，单击右上角 **结束任务** 按钮。
    
    ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5zaMjPMQnKa/img/a2364598-45b0-4184-9ed9-17133a21350c.png)
    
2.  系统弹出红色警告样式的确认对话框："确定结束任务？结束后不可恢复，未开始升级的设备将标记为已终止。"
    
3.  单击 **确定**：
    
    *   立即停止向未开始升级的设备下发升级指令。
        
    *   所有状态为"待升级"的设备标记为"已终止"。
        
    *   已开始升级（状态为"升级中"）的设备继续完成当前升级。
        
4.  任务状态变为"已结束"，操作按钮隐藏，自动刷新停止。
    
5.  页面自动刷新任务状态、进度统计和设备列表数据。
    

:::
**注意**：

结束操作不可恢复。如需重新升级，请创建新的升级任务。

操作执行过程中，按钮显示 loading 状态，防止重复点击。

在确认对话框中单击"取消"，关闭对话框，不执行任何操作。
:::

### 任务状态流转图

```plaintext
                                                    ┌──全部完成──▶ 已完成(COMPLETED)
                                                    │
提交发布 ──▶ 待审批(PENDING_APPROVAL) ──审批通过──▶ 待执行(PENDING) ──到达开始时间──▶ 执行中(RUNNING)
  │              │                                  │                             │
  │              └──审批驳回──▶ 已驳回(REJECTED)      └──结束任务──▶ 已结束(ENDED)  ◀──┘
  │
  └──（手动导入策略，无需审批）──▶ 待执行(PENDING) ──到达开始时间──▶ 执行中(RUNNING) ...

```

*   `PENDING_APPROVAL` → `PENDING`：产线负责人审批通过（仅指定版本号升级和文件导入升级策略）。
    
*   `PENDING_APPROVAL` → `REJECTED`：产线负责人审批驳回，任务终止。
    
*   `PENDING` → `RUNNING`：到达任务周期的开始时间，系统自动开始执行。
    
*   `RUNNING` → `COMPLETED`：所有设备升级完成（成功或失败），或任务周期结束时间到达。
    
*   `PENDING` → `ENDED`：运维人员在任务开始前结束任务。
    
*   `RUNNING` → `ENDED`：运维人员在任务执行过程中结束任务。
    
*   **已完成**、**已结束** 和 **已驳回** 为终态，不可再流转。

## 8 常见问题（Q&A）

**Q1：整包升级和差分升级应该怎么选？**

:::
**整包升级** 适用于所有产线，兼容性好，支持跨多个版本升级，是默认推荐的升级方式。

**差分升级** 目前仅 **4G 模组**产线支持，包体积小、传输快，适合 4G 模组的小版本迭代场景。如果升级固件不属于 4G 模组产线，请选择整包升级。
:::
---

**Q2：选择差分升级后，目标固件版本下拉列表为空怎么办？**

:::
说明当前没有可用的差分包。请前往 **固件管理 > 差分管理**，为目标版本和待升级的基线版本生成差分包。差分包生成成功后，返回新增任务页面即可选择。也可以改用整包升级。
:::
---

**Q3：为什么"预览并发布"按钮无法点击？**

:::
**Step 1**：确认任务名称、升级策略、升级包类型、目标固件版本和任务所属大区均已填写，且任务名称未重复。

**Step 2 指定版本号升级**：确认已勾选至少一个固件版本。

**Step 2 文件导入升级**：确认文件已上传且存在有效设备。

**Step 2 手动导入升级**：确认已输入设备标识并执行了校验操作，且存在有效设备。
:::
---

**Q4：为什么部分设备显示"不可升级"？**

:::
常见原因包括：

**设备离线**：设备当前不在线，无法接收升级指令。

**已是目标版本**：设备当前固件版本与目标版本相同。

**固件不兼容**：设备硬件版本与目标固件包不兼容。

**无可用差分包**：差分升级时，设备当前版本与差分包基线版本不匹配。

**设备不存在**：输入的设备标识在平台中未注册。
:::
---

**Q5：差分升级时，部分设备提示"无可用差分包"怎么办？**

:::
说明这些设备的当前固件版本没有对应的差分包。处理方式：

前往 **固件管理 > 差分管理**，为这些设备的当前版本生成差分包。

或者改用整包升级，整包升级不受设备当前版本限制，且支持所有产线。

也可以先对有差分包的设备执行差分升级，再单独为剩余设备创建整包升级任务。

如果设备不属于 4G 模组产线，则不支持差分升级，请直接使用整包升级。
:::
---

**Q6：结束任务后，正在升级中的设备会怎样？**

:::
结束任务仅停止向未开始升级的设备下发升级指令。已开始升级（状态为"升级中"）的设备会继续完成当前升级，最终状态为"升级成功"或"升级失败"。
:::
---

**Q7：结束任务后还能恢复吗？**

:::
不能。结束操作不可恢复。如需重新升级，请创建新的升级任务。
:::
---

**Q8：任务详情页的数据多久刷新一次？**

:::
任务处于"执行中"状态时，每 15 秒自动刷新。其他状态不自动刷新。可随时单击"手动刷新"按钮获取最新数据。
:::
---

**Q9：单次升级任务最多支持多少台设备？**

:::
**指定版本方式**：无明确上限，取决于设备数量。

**文件导入方式**：单个文件最多 20,000 个设备ID。

**手动导入方式**：单次最多 10 个设备标识。
:::
---

**Q10：升级失败的设备如何处理？**

:::
在任务详情页中，通过状态筛选定位升级失败的设备，查看"失败原因"列。

**常见处理方式**：

检查设备网络连接是否正常。

确认设备硬件是否兼容目标固件。

排查问题后，创建新的升级任务对失败设备重新升级。
:::
---

**Q11：任务周期有什么限制？**

:::
开始时间必须晚于当前时间，不能设置过去的时间。

任务周期（结束时间 - 开始时间）不能超过 7 天。

任务到达开始时间后自动开始执行，到达结束时间后自动停止下发新的升级指令。

建议根据设备数量和网络条件合理设置周期，避免周期过短导致部分设备未能完成升级。
:::
---

**Q12：为什么提交任务后状态是"待审批"而不是"待执行"？** 

:::
选择"指定版本号升级"或"文件导入升级"策略时，任务提交后需经产线负责人审批通过后才能执行。这是因为这两种策略涉及批量设备升级，需要产线负责人确认升级范围和影响。选择"手动导入升级"策略时，任务提交后直接进入"待执行"状态，无需审批。
:::
---

**Q13：审批被驳回后怎么办？** 

:::
审批驳回后，任务状态变为"已驳回"，您可以在任务详情页查看驳回原因。如需重新升级，请根据驳回原因调整配置后创建新的升级任务。已驳回的任务不可修改或重新提交。
:::
